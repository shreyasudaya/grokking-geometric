# Grokking-Geometric — Reproducibility Notes

Grokking baselines for multiple mathematical operations: modular addition/subtraction/multiplication, symmetric group (S₅) composition, and permutation (S₆) composition.

## Prerequisites
- Python 3.10+ and the versions in `requirements.txt` (recommended: `torch>=2.1.0`).
- CUDA + appropriate PyTorch build if you want GPU/bfloat16 runs.

For fresh-clone setup, full-sweep commands, resume instructions, storage
expectations, and what to commit, see [`RUNNING.md`](RUNNING.md).

## Datasets

Each dataset is generated by running its `prepare.py` script:

```bash
python data/modular_addition/prepare.py           # (a + b) mod 97
python data/modular_subtraction/prepare.py        # (a - b) mod 97
python data/modular_multiplication/prepare.py     # (a * b) mod 97
python data/symmetric_group/prepare.py            # S₅ composition
python data/permutation_composition/prepare.py    # S₆ composition
```

- Modular operations: 97² = 9409 equations, 25% train / 75% val.
- S₅: 120² = 14400 equations, 25% train / 75% val.
- S₆: 720² = 518400 equations, 25% train / 75% val.

## Training

For new experiments and sweeps, prefer the Hydra runner because it has storage
guards enabled by default:

```bash
python run.py dataset=modular_addition model.n_layer=2 model.n_embd=128 seed=0
```

Default storage policy:
- saves checkpoints every 200 steps, not every 10 steps;
- saves model weights only, without optimizer/RNG state;
- analyzes every saved checkpoint;
- deletes analyzed checkpoints after `signals.csv` is written;
- prunes a run if it exceeds `storage.max_run_gb` (default: 9.5 GB).

To keep checkpoints for debugging, opt in explicitly:

```bash
python run.py storage.keep_checkpoints_after_analysis=true storage.save_optimizer=true storage.save_rng=true
```

For a fresh storage-safe sweep, run:

```powershell
.\run_sweeps.ps1
```

This writes to a timestamped `runs_fresh_*` directory and aggregates results
under `results/<run-root>/`.

Select a dataset with `--dataset`:

```bash
# Full-batch reproducible run
python train.py --dataset modular_addition --full-batch --disable-fused --deterministic --seed 1337 --max-iters 50000 --eval-interval 100

# Quick smoke test
python train.py --dataset symmetric_group --disable-fused --max-iters 20 --eval-iters 2 --eval-interval 5
```

### CLI Arguments

| Argument | Default | Description |
|----------|---------|-------------|
| `--dataset` | `modular_addition` | Dataset: modular_addition, modular_subtraction, modular_multiplication, symmetric_group, permutation_composition |
| `--n-layer` | 2 | Number of transformer layers |
| `--n-head` | 4 | Number of attention heads |
| `--n-embd` | 128 | Embedding dimension |
| `--max-iters` | 50000 | Total training steps |
| `--eval-interval` | 100 | Checkpoint save frequency |
| `--eval-iters` | 50 | Batches for loss estimation |
| `--full-batch` | off | Use entire training set as one batch |
| `--seed` | 1337 | Random seed |
| `--deterministic` | off | Deterministic CUDA ops |
| `--disable-fused` | off | Disable fused AdamW |
| `--out-dir` | out-grokking | Output directory |

### Default Model Architecture
- **~0.4M parameters** (2 layers, 4 heads, 128 embedding)
- Zero dropout, no bias in LayerNorm/Linear
- Weight-tied embeddings
- Flash attention when available

Override architecture:
```bash
python train.py --dataset modular_addition --n-layer 4 --n-head 4 --n-embd 256 --max-iters 100000
```

### What is logged/saved
- `{out_dir}/run_metadata.json`: environment and flags used for the run.
- Checkpoints: `{out_dir}/ckpt_{iter}.pt` — each checkpoint includes `model`, `optimizer`, `model_args` (with `block_size`, `vocab_size`), `dataset_name`, `n_output`, `iter_num`, `best_val_loss`, `rng_states`, and `run_metadata`.

## Post-Training Analysis

Note: the legacy `train.py` path still writes full checkpoints. Use sparse
`--save-every` / `--eval-interval` values if you run it directly, or use
`run.py` for the 10 GB storage policy.

```bash
python plot_and_analyze.py --out-dir out-grokking --eval-iters 5 --batch-size 256
```

Generates loss plots, detects phase transition, and runs Hessian-based geometric analysis.

## Reproducibility tips
- Use `--disable-fused` to avoid fused optimizer numerical differences across PyTorch builds.
- Use `--deterministic` to force deterministic algorithms.
- Match hardware precision — set `dtype` in `train.py` if necessary.
- To resume exactly, load RNG states from checkpoint and restore them before continuing.
